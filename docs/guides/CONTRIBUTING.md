## 1. Измените статус задачи в [Trello](https://trello.com/b/dVBBwC4t/%D0%B7%D0%B0%D0%BC%D0%BE%D1%80%D1%8B%D1%88)

Переместите карточку задачи в колонку `In Progress`.

## 2. Синхронизируйтесь с GitHub и создайте новую ветку

1. Перед началом работы в терминале убедитесь, что вы находитесь в директории проекта. Если вы находитесь в родительской
   папке, перейдите в директорию проекта:
   
    ```bash
    cd zamorysh
    ```
    
2. Перейдите на ветку main и синхронизируйтесь с GitHub:

    ```bash
    git checkout main
    git pull
    ```
    
3. Создайте новую ветку для задачи, перейдите на нее:

    ```bash
    git checkout -b feature-name
    ```

## 3. Работайте в этой ветке

1. Добавьте файлы в коммит (можно все, можно только те, изменения в которых вы хотите сохранить):

    ```bash
    git add .
    ```
    
    ```bash
    git add path/to/file.py
    ```
    
2. Сделайте коммит. Описание коммита должно быть коротким предложением на английском языке и начинаться с глагола в
   императиве.
   
    ```bash
    git commit -m "Add commit name here"
    ```
    
3. Повторяйте, пока не закончите работу над задачей или пока у вас не возникнут вопросы.

## 4. Говорите о проблемах

### Если есть вопросы

Если ваш код не запускается или падает во время выполнения, сначала попытайтесь разобраться сами. Посмотрите на
сообщение об ошибке, поищите ответ на StackOverflow, спросите нейросеть. Если ничего из этого не помогает, задайте
вопрос.
Если вы совсем не понимаете, с чего начать и что делать, задайте вопрос.
Задать вопрос можно двумя способами: можно написать мне в личку, можно написать в чат. Перед тем, как задать вопрос,
подумайте, может ли ответ быть полезен другим участникам, и действуйте в соответствии с этим. Я постараюсь ответить на
вопрос в течение 24 часов.

### Если вы не успеваете

В этом нет ничего страшного. Возможно, у задачи слишком расплывчатое ТЗ или задача слишком объемная. Пожалуйста,
сообщайте мне об этом, как только поймете.
Если вы успеваете сделать что-то, то сделать что-то лучше, чем не сделать ничего. В таком случае можно не создавать pull
request, но всё равно нужно будет рассказать на встрече, что вы успели сделать.

### Если ваша задача состоит из нескольких этапов

Например, сначала Вам нужно придумать, как что-то реализовать, а потом написать код. В таком случае лучше сообщайте мне
о том, что вы сделали, после каждого этапа, чтобы в случае чего не тратить время зря.

## 5. Протестируйте новые функции

Перед тем как создавать pull request, проверьте, что всё работает так, как вы задумывали. Запустите игру, включите
логирование, протестируйте разные случаи.

## 6. Создайте pull request

1. Перейдите на ветку main и обновите ее:

    ```bash
    git checkout main
    git pull
    ```
    
2. Перейдите на рабочую ветку и влейте в нее обновленную ветку main:

    ```bash
    git checkout feature-name  
    git merge main
    ```
    
3. При наличии конфликтов разрешите их и еще раз проверьте, что всё работает.
4. Отправьте ветку на GitHub:

    ```bash
    git push -u origin feature-name
    ```
    
5. В терминале появится ссылка для создания pull request. Перейдите по ней, напишите название и описание при
   необходимости, создайте pull request.

## 7. Измените статус задачи в [Trello](https://trello.com/b/dVBBwC4t/%D0%B7%D0%B0%D0%BC%D0%BE%D1%80%D1%8B%D1%88)

Переместите карточку задачи в колонку `Done`.

## 8. Дождитесь merge

Если у меня будут какие-то замечания, я оставлю на странице pull request комментарии и сообщу вам об этом. После merge я
поставлю в Trello галочку рядом с задачей.

## 9. Удалите ветку из локального репозитория

```bash
git branch -d feature-name
```

Из репозитория на GitHub ветка будет удалена автоматически.

На встрече расскажите о том, что именно вы сделали, покажите и разберите свой код.
